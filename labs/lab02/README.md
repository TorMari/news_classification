## 3. Основні правила очистки та нормалізації

- Маскування: `<URL>`, `<EMAIL>`, `<PHONE>`  
- Видалення блоків із фото- та соц-кредитами  
- Нормалізація лапок (`“”` → `"`, `‘’` → `'`)  
- Нормалізація дефісів (`–`, `—`, `−` → `-`)  
- Заміна NBSP на звичайний пробіл, очистка зайвих пробілів  
- Sentence split з урахуванням скорочень (`м.`, `ім.`, `т.д.`, `р.`, `т.п.`, `дол.`, `тис.`) та чисел (`9.00`, `15.7`)

---

## 4. 5 найболючіших edge cases у датасеті
   
1. Не розбивати речення після порядкових номерів у списках (`1./2./... `)
2. Не розбивати на два речення числові значення з крапкою `+5.23,5`  
3. Стандартний sentence splitter розривав речення після скороченнь `м.`, `р.`, `дол.`  
4. Фото-/соц-кредити у різних форматах (`Фото: ...`, `Допис ...(@...)`)  
5. У `(https://...)` не видаляти дужки при заміні на URL

---

## 5. Що стало краще після

- Нормалізовані тире, дефіси, лапки, переноси, табуляції та нерозривні пробіли
- Середня довжина текстів зменшилася (`1406.56` → `1396.6`), видалено незмістовні речення 
- Завдяки sentence split речення правильно відокремлено з урахуванням скорочень, абревіатур та чисел
