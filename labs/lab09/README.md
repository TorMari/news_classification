---

## 3. Які параметри використано

Параметри: vector_size=100, window=5, min_count=3

Вибір архітектури: sg=1 (Skip-gram)

Пояснення: Skip-gram краще працює з рідкісними словами та невеликими корпусами, оскільки намагається передбачити контекст за словом, а не навпаки.

## 4. Які типи слів аналізували

У ході аналізу розглядалися такі типи слів:

frequent — частотні слова (наприклад: фільм)
rare — рідкісні слова (саундтрек)
domain — доменні терміни (енергосистеми, санкції)
morph-variant — словоформи (дискваліфікація, оприлюднений/оприлюднити)
noisy / named entities — імена, трансліт (Phillip Nova)

---

## 6. Де FastText був кращий / де ні

* FastText кращий: морфологічні варіанти, рідкісні слова, трансліт та noisy дані
* FastText не дає переваги: дуже рідкісні слова без контексту, шумні або випадкові слова, вузькоспеціалізовані терміни з малою частотою
* Word2Vec кращий: частотні слова, чисті семантичні зв’язки, менш шумні результати

---

## 7. Чи embeddings корисні для цього корпусу

Embeddings є корисними для аналізу новинного корпусу, оскільки дозволяють виявляти семантичні зв’язки та тематичні залежності між словами. FastText показав кращі результати для української мови завдяки здатності обробляти морфологію та рідкісні слова. Водночас якість embeddings обмежується невеликим розміром корпусу та наявністю шуму. Для покращення результатів доцільно збільшити обсяг даних і покращити preprocessing. У цілому embeddings дають корисний сигнал, але потребують оптимізації для більш точних результатів.
