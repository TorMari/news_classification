# Lab 2 — Lemma/POS baseline (UA)

## 1. Задача

**Тип задачі:** А — класифікація текстів (тема/тональність/спам/urgency)  

---

## 2. Інструмент Lemma/POS
Для проведення лематизації та формування послідовністі POS-тегів використовулися інструменти бібліотеки `Stanza`.

---

## 3. Baseline

Для задачі класифікації українських новин було реалізовано два базові підходи:
* TF-IDF + Logistic Regression 
* TF-IDF + Linear SVM

---

## 4. Метрики “до/після” лематизації

## Baseline: TF-IDF + Logistic Regression
* Accuracy до: 0.9583
* Accuracy після: 0.9524
* Macro-F1 до: 0.9580
* Macro-F1 після: 0.9524

## Baseline: TF-IDF + Linear SVM
* Accuracy до: 0.9583
* Accuracy після: 0.9583
* Macro-F1 до: 0.9580
* Macro-F1 після: 0.9581

---

## 5. Висновок

Лематизована версія тексту lemma_text не продемонструвала покращення метрик (Accuracy та Macro-F1) порівняно з processed_v2. У межах даного корпусу обидва baseline-підходи (Logistic Regression та Linear SVM) показали майже однакову ефективність, однак стабільно кращі результати були отримані для версії processed_v2.
Отже, для зібраного корпусу та задачі класифікації використання сирого processed_v2 є доцільнішим, оскільки лематизація не забезпечує додаткового приросту якості та може призводити до втрати інформаційно значущих елементів тексту.
