# Lab 4 —  Rule-based IE (regex + dictionaries)

## 1. Типи полів, що витягуються

У цій роботі реалізовано витягування трьох типів полів з текстового корпусу:
* **DATE** — календарні дати подій (наприклад: 12 лютого, 5 травня 2024).
* **AMOUNT** — грошові суми або фінансові значення (наприклад: 100 грн, 200 доларів).
* **SCORE** — спортивний рахунок між двома командами або учасниками (наприклад: 2:1, 3-0).

---

## 2. Використані правила та словники

Для витягування полів використано регулярні вирази та прості контекстні фільтри:
* Regex-патерни для пошуку дат, сум та рахунків.
* Список місяців українською мовою для розпізнавання дат.
* Контекстні фільтри для виключення неправильних спрацювань (наприклад, час 15:00 не вважається рахунком).

## 4. Топ-5 edge cases у корпусі

1. "12 та 14 травня 2026" — у тексті згадано дві дати, але regex може сприймати це як одну.
2. "7 липня о 18:00" — дата присутня, але час не повинен витягуватися як SCORE.
3. "у більшості зон світло є 3-4 години" — числовий діапазон може бути помилково визначений як рахунок.
4. "Netflix займає 9% телевізійного бізнесу" — відсоток не є грошовою сумою.
5. "у 2023-2024 роках" — діапазон років не є спортивним рахунком.

## 5. Подальші покращення

У наступних лабораторних роботах планується:
* покращити правила фільтрації контексту для зменшення false positives;
* розширити набір правил для обробки складних конструкцій у тексті;
* використати більш складні шаблони для витягування інформації.
