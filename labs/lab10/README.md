# Lab 10 — NER pipeline + hybrid rule

## Evaluation set
Використано evaluation set із 26 речень, відібраних з основного корпусу та розмічених вручну. Поля кожного запису: `id`, `text`,`expected_entities`, `comment`.

## Pipeline

Застосовано гібридний підхід: baseline NER-модель на основі `Stanza` із подальшою постобробкою, що включає regex-правила та domain-specific словники.

## Додані rules:
1. regex для `DATE` та `MONEY`
2. merge-правило для об’єднання `PERSON`
3. словник доменних організацій (`ORG`)

## Що baseline знаходив добре
Модель добре розпізнавала `PERSON` (precision/recall = 1.00) та демонструвала стабільні результати для `LOC`. Частково коректно визначались `ORG`.

## Які сутності baseline пропускав
Повністю пропускались часові сутності та суми грошей (`DATE`, `MONEY`), а також частина доменних організацій (`ORG`), що знижувало recall.

## Що rules покращили:
* `DATE` (regex): модель почала коректно розпізнавати дати, зокрема формати на кшталт «5 грудня», «17 січня 2026 року»;
* `MONEY` (regex): успішно витягуються грошові значення, наприклад «82.7 мільярда доларів», «11 мільйонів гривень»;
* `PERS` (merge rule): об’єднання часткових сутностей дозволило відновлювати повні імена (наприклад, «Трамп» -> «Дональд Трамп»);
* `ORG` (domain-specific rules): додання словника доменних організацій (БЕБ, ЮНІДО, Netflix тощо) покращило їх розпізнавання.

* суттєве зростання recall для `DATE` і `MONEY` (до 1.00)
* повне покриття ORG (recall = 1.00)
* покращення повноти витягування загалом

## Невирішені проблеми:
* наявність хибнопозитивних спрацювань (false positives) для типу `LOC` та `ORG`:
* окремі слова, такі як «Кишинів», «Арені» або «Екзорцисті» (назва фільму), іноді класифікуються як географічні об’єкти або організації без урахування контексту.
