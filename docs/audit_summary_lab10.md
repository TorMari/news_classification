# Audit Summary — Lab 10 (NER pipeline + hybrid rule)
## Використаний pipeline
Застосовано гібридний підхід: baseline NER-модель на основі `Stanza` із подальшою постобробкою, що включає regex-правила та domain-specific словники.

## Ключові сутності задачі
`LOC`, `ORG`, `PERSON`, `DATE`, `MONEY` — основні типи сутностей, релевантні для новинного корпусу.

## Що baseline знаходив добре
Модель добре розпізнавала `PERSON` (precision/recall = 1.00) та демонструвала стабільні результати для `LOC`. Частково коректно визначались `ORG`.

## Які сутності baseline пропускав
Повністю пропускались часові сутності та суми грошей (`DATE`, `MONEY`), а також частина доменних організацій (`ORG`), що знижувало recall.

## Додані правила
1. regex для `DATE` та `MONEY`
2. merge-правило для об’єднання `PERSON`
3. словник доменних організацій (`ORG`)

## Покращення завдяки використанню правил
1. суттєве зростання recall для `DATE` і `MONEY` (до 1.00)
2. повне покриття `ORG` (recall = 1.00)
3. покращення повноти витягування загалом

## Наймасовіші категорії помилок
1. missed entities (`DATE`, `MONEY`, `ORG` — до правил)
2. boundary errors (`PERSON`)
3. false positives для `LOC` та частково `ORG`

## Подальші покращення
Додати контекстні правила або класифікатор для зменшення false positives. Також розширити domain-словники для назв організацій.
Крім того, можна впровадити оцінку F1 та здійснити аналіз помилок на більшій вибірці або повному датасеті.
