# NER Pipeline Summary (Stanza-based)

## 1. Обраний pipeline

Використано NER-пайплайн на основі `Stanza` з подальшою гібридною постобробкою.

## 2. Сутності “з коробки”

Модель підтримує базові типи сутностей: `PERSON`, `ORG`, `LOC`, `DATE`, `MONEY`.

## 3. Важливі доменні сутності

Для новинного корпусу критичними є:
* організації (державні органи, міжнародні інституції, компанії),
* географічні назви (`LOC`)

## 4. Додані правила

* regex для `DATE` і `MONEY`
* merge-логіка для повних імен `PERSON`
* словник доменних організацій (абревіатури та специфічні назви)

## 5. Покращення після правил

* `DATE` (regex): модель почала коректно розпізнавати дати, зокрема формати на кшталт «5 грудня», «17 січня 2026 року»;
* `MONEY` (regex): успішно витягуються грошові значення, наприклад «82.7 мільярда доларів», «11 мільйонів гривень»;
* `PERS` (merge rule): об’єднання часткових сутностей дозволило відновлювати повні імена (наприклад, «Трамп» -> «Дональд Трамп»);
* `ORG` (domain-specific rules): додання словника доменних організацій (БЕБ, ЮНІДО, Netflix тощо) покращило їх розпізнавання.

* суттєве зростання recall для `DATE` і `MONEY` (до 1.00)
* повне покриття ORG (recall = 1.00)
* покращення повноти витягування загалом

## 6. Залишкові помилки

* наявність хибнопозитивних спрацювань (false positives) для типу `LOC` та `ORG`:
* окремі слова, такі як «Кишинів», «Арені» або «Екзорцисті» (назва фільму), іноді класифікуються як географічні об’єкти або організації без урахування контексту.

## 7. Подальші покращення

* додавання контекстних фільтрів для зменшення false positives
* розширення словників і правил
* донавчання моделі на доменному корпусі
